# Протокол сообщений phone ↔ watch

Транспорт: Connect IQ Mobile SDK (Android `sendMessage`) ↔ Monkey C `Communications` mailbox.
Каждое сообщение — `Dictionary`. Принцип: маленькие сообщения, stop-and-wait с подтверждением.

## Кодирование координат

- Градусы → **целые микроградусы**: `mdeg = round(deg * 1_000_000)`.
- В чанках линии передаём **дельты** от предыдущей точки (первая точка — абсолютная).
  Соседние точки близки → дельты маленькие → меньше байт.
- Восстановление на часах: накопительная сумма дельт.

## Типы сообщений

### `H` — Header (phone→watch)
```
{ "t":"H", "v":1, "np":<int точек линии>, "nc":<int чанков>,
  "nm":<int манёвров>, "name":<string> }
```
Часы: сбросить буфер, выделить хранилище, перейти в RECEIVING. Ответ: `A(-1)`.

### `L` — Line chunk (phone→watch)
```
{ "t":"L", "s":<seq 0..nc-1>,
  "p":[[dlat,dlon],[dlat,dlon], ...] }   // микроградусы, дельты
```
Первый элемент чанка 0 — абсолютные координаты старта, далее дельты. Между чанками дельта
считается от последней точки предыдущего чанка. Ответ: `A(s)`.

Рекомендация: `POINTS_PER_CHUNK ≈ 40` (подобрать на железе под лимит сообщения).

### `M` — Maneuvers (phone→watch)
```
{ "t":"M", "m":[[idx,type,dist], ...] }
```
- `idx` — индекс точки в *децимированной* линии (0-based).
- `type` — `0` прямо/слияние, `±1` плавно, `±2` обычный, `±3` резкий (знак: + право, − лево),
  `9` финиш.
- `dist` — накопленная дистанция от старта до манёвра, метры (int).

Обычно весь список влезает в одно сообщение. Если манёвров много — разрешено слать несколько
`M` подряд (каждый со своим ack `A(-2)`); часы аппендят.

### `E` — End (phone→watch)
```
{ "t":"E" }
```
Часы проверяют: собрано ровно `np` точек и `nm` манёвров. Ок → `A(-3)`, состояние READY.
Не ок → `N(seq первого недостающего чанка)`.

### `A` — Ack (watch→phone)
```
{ "t":"A", "s":<seq | -1 H | -2 M | -3 E> }
```

### `N` — Nack (watch→phone)
```
{ "t":"N", "s":<seq для повтора> }
```

## Поток (stop-and-wait)

```
phone: H ───────────▶
                      ◀─────────── A(-1)   watch
phone: L0 ──────────▶
                      ◀─────────── A(0)
phone: L1 ──────────▶
                      ◀─────────── A(1)
        ...
phone: M ───────────▶
                      ◀─────────── A(-2)
phone: E ───────────▶
                      ◀─────────── A(-3)   → READY
```

Таймаут ожидания ack: `ACK_TIMEOUT = 4 с`. Ретраев на пакет: `MAX_RETRY = 3`. После
исчерпания — ошибка в UI телефона с кнопкой Retry (рестарт с `H`).

## Прикидка размеров (ориентир, выверить на устройстве)

- Точка как `[dlat,dlon]` — две небольшие целые. На дистанции навигации дельта обычно
  ≤ несколько тысяч микроградусов.
- ~40 точек на чанк держат сообщение в безопасной зоне относительно лимита mailbox
  (фоновые данные у CIQ ~8 КБ, но целимся существенно ниже).
- Манёвров на типичный городской маршрут — десятки, один `M` обычно достаточно.

`POINTS_PER_CHUNK` и `MAX_LINE_POINTS` — конфигурируемые; их финальные значения
определяются реально измеренным лимитом сообщения на Instinct 3 (`// TODO: verify on device`).

## Версионирование

Поле `v` в `H`. При несовпадении версий часы показывают «Update app» и игнорируют передачу.
Любое изменение схемы → инкремент `v` + правка этого файла + синхронная правка обеих сторон.
