# CLAUDE.md — инструкции для Claude Code

Проект **my_routecast**: Android-компаньон + Connect IQ виджет для передачи маршрута из
Organic Maps на Garmin Instinct 3 Tactical и отрисовки линии + псевдо-TBT. Полное ТЗ — в
`SPEC.md`, протокол — в `docs/protocol.md`. Этот файл — про то, КАК вести работу в репозитории.

## Контекст и неизменяемые рамки

- Источник маршрута — **только Organic Maps** (GPX/GeoJSON). Не добавлять Google Maps.
- На часах нет базовой карты и нет нативной навигации — всё рисуем сами.
- Передача через **Connect IQ Mobile SDK поверх Garmin Connect Mobile**. Не пытаться слать
  файлы на часы напрямую по BLE мимо GCM — открытого протокола для этого нет.
- Виджет открывается **вручную перед шарингом**. Не закладывать «мгновенное пробуждение
  аппа с телефона» — в CIQ его нет; не тратить время на обходные пути.
- Сообщения CIQ маленькие → **децимация + чанкинг обязательны**, протокол менять только
  синхронно с обеих сторон и с `docs/protocol.md`.
- **Симулятор первым.** Ничего не ставится на реальные часы/телефон, пока не пройдены гейты
  симуляции из `docs/testing.md` (Gate 1 юнит-тесты Android → Gate 2 CIQ Simulator → Gate 3
  adb-tethered). Железо (Gate 4) — финальная стадия, только для того, что симулятор не
  воспроизводит. Каждую фичу проектировать так, чтобы её можно было проверить в симуляторе
  до железа. Не предлагать установку на устройство как способ «быстрее проверить».

## Порядок работы (рекомендуемая последовательность)

1. **Watch-скелет:** виджет с mailbox-listener, который принимает чанки по протоколу и
   логирует сборку маршрута (без рендера). Проверить на симуляторе с моками сообщений.
2. **Android-скелет:** intent-filter + парсер GPX → список точек → лог. Без передачи.
3. **Передача:** децимация + манёвры + stop-and-wait + ack. Сначала отладить на симуляторе
   (adb forward), затем на железе.
4. **Рендер на часах:** проекция → линия heading-up → «я» + стрелка.
5. **Псевдо-TBT:** баннер поворота, дистанция, off-route, вибро.
6. **Полировка:** retry-UI, прогресс, граничные случаи (пустой/однотрековый GPX, очень
   длинный маршрут, потеря GPS).

Делать вертикальными срезами: каждый шаг должен давать что-то проверяемое **в симуляторе**,
а не «весь Android, потом весь watch». Порядок гейтов проверки и сценарии — в `docs/testing.md`;
держать его в зелёном по мере реализации, новые фичи сопровождать симуляционными проверками.

## Конвенции

- Координаты в протоколе — **целые микроградусы** (`deg*1e6`), дельты между точками.
- Все магические числа (`MAX_LINE_POINTS`, `POINTS_PER_CHUNK`, `TURN_MIN`, `OFFROUTE_M`,
  таймауты) — в одном конфиг-месте на каждой стороне, не разбрасывать по коду. Их предстоит
  крутить на железе.
- UUID watch-аппа — единый источник истины: `watch/manifest.xml`. На Android — продублировать
  в `Constants`/`BuildConfig`, не хардкодить в разных местах.
- Никаких сетевых вызовов, аналитики, телеметрии. Оффлайн-only, приватность по умолчанию.
- Watch-код: минимум аллокаций в `onUpdate`, тяжёлое (проекция всех точек) — кешировать и
  пересчитывать только при смене позиции/heading/зума, не каждый кадр.

## Чего НЕ делать без явного запроса

- Не публиковать в Connect IQ Store, не добавлять submit-флоу.
- Не добавлять обязательные онлайн-зависимости/ключи API.
- Не менять формат входа на «ссылку Google Maps» или онлайн-роутинг — это сознательно
  исключено в `SPEC.md`.
- Не расширять протокол молча — сначала правка `docs/protocol.md`, потом обе стороны.

## Сборка / запуск

**Watch:** Monkey C extension (VS Code) → "Build for Device". Device id Instinct 3 искать в
`<sdk>/bin/devices.xml` — **не выдумывать**. Выход `.prg` → `\GARMIN\APPS\` по USB.

**Android:** стандартный Gradle build → `adb install`. Зависимость
`com.garmin.connectiq:ciq-companion-app-sdk` с Maven Central.

## Открытые вопросы под железо (см. SPEC §7)

Размер сообщения mailbox, стабильность BLE/таймауты, точный device id, доступность heading,
монтирование как mass storage. Эти значения — placeholder'ы в конфиге; помечать `// TODO: verify on device`.

## Заметки по окружению автора

Репозиторий живёт в `D:\Github\my_routecast\`. Часы подключаются как накопитель, привычный
путь для файлов — `GARMIN\NewFiles\` (для `.prg` использовать `GARMIN\APPS\`). Garmin Connect
Mobile уже установлен и часы спарены.
